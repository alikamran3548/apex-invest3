Build me a full-stack web application called "APEX INVESTOR" — a comprehensive personal investment learning, tracking, analysis, and implementation platform. This is my life's investing companion app. It must be beautifully designed, blazing fast, fully structured, and dead simple to navigate despite its depth.

## TECH STACK

- Frontend: Next.js 14 (App Router) with TypeScript
- Styling: Tailwind CSS + shadcn/ui component library
- Database: SQLite via Prisma ORM (local-first, no external DB needed)
- State: Zustand for global state
- Storage: All data persists locally in SQLite database file
- Search: Built-in full-text search across all sections
- Export: JSON export/import for backup
- No authentication needed — this is a personal single-user app

## DESIGN & UI REQUIREMENTS

- Dark mode by default (with light mode toggle)
- Color scheme: Deep navy/charcoal background, gold/amber accents for S-tier highlights, green for profits, red for losses
- Left sidebar navigation with collapsible category groups and icons for every section
- Each section opens in the main content area (no page reloads, smooth transitions)
- Every section has: a title header, description subtitle, an "Add Entry" button, and entries displayed as expandable cards
- Cards should show preview text, date created, tags, and expand on click to show full content
- Rich text editor (use Tiptap) for all text entry fields so I can format notes with bold, italic, bullet points, headers, highlights
- Global search bar at the top that searches across ALL sections
- "Quick Add" floating button (bottom-right) that lets me pick any section and add an entry from anywhere
- Dashboard home page showing: total entries count per section, recent activity feed, pinned/starred items, and a motivational investing quote
- Responsive design but optimized for desktop use
- Smooth animations (framer-motion) for card expansions, page transitions, sidebar toggle
- Every entry everywhere should have: created date, last modified date, tags (custom), star/pin toggle, archive toggle
- Drag and drop reordering within sections

## DATABASE SCHEMA

Design the Prisma schema to support all sections below. Use a flexible but structured approach:

- A `Section` model (id, name, slug, category, icon, description, sortOrder)
- An `Entry` model (id, sectionId, title, content (rich text JSON), tags[], starred, archived, metadata (JSON field for section-specific structured data), createdAt, updatedAt)
- A `Tag` model (id, name, color)
- A `Portfolio` model (id, ticker, companyName, entryPrice, currentPrice, shares, entryDate, exitDate, exitPrice, status (holding/sold/watching), entryReason, holdingReason, sellReason, returns, notes, sector, country)
- A `StockWatchlist` model (id, ticker, companyName, sector, country, addedDate, reason, targetPrice, currentPrice, status, notes)
- A `QuarterlyAnalysis` model (id, quarter, year, ticker, companyName, analysis, experience, keyMetrics JSON, createdAt)
- A `CompanyAnalysis` model (id, ticker, companyName, advantages, disadvantages, secretActivities, corrections, recommendations JSON array, sector, country, updatedAt)
- A `MarketAnalysis` model (id, title, marketCondition, economyCorrelation, analysis, date, indicators JSON)
